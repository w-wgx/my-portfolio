### 6. 意外干扰检查
属性 是否存在 影响分析 transform ❌ 无 - opacity ❌ 无 - z-index ✅ z-10 ASCIIText 和内容层都用了 z-10 ，但背景层有 z-0 包裹，理论上内容层应该在上方 overflow: hidden ✅ section 可能导致问题 ：section 是 h-screen ，如果 ASCII 文本超出视口会被裁剪 mix-blend-mode ✅ difference 可能导致问题 ：在浅色主题下 mix-blend-mode: difference 可能显示异常

潜在问题：

1. mix-blend-mode: difference （ ASCIIText.tsx:145 ）：在浅色主题下，白色背景与浅色文本混合可能导致对比度问题
2. h-screen + overflow-hidden ：当页面滚动时，背景层不会跟随滚动，可能导致视觉断层
3. 双重绝对定位 ：ASCIIText 组件内部的 position: absolute 与父容器的 absolute inset-0 重叠，可能导致尺寸计算异常
4. z-10 冲突 ：ASCIIText 和主内容区都使用了 z-10 ，虽然背景层有 z-0 包裹，但在某些情况下可能出现层级混乱
### 7. 修复建议
问题1：双重绝对定位

问题2：浅色主题下 mix-blend-mode 显示异常

```
  适用 */
  /* 浅色主题可能需要改为 normal 或其他模式 
  */
}
```
